Current Project

Reach 11 Recreational Park: Trail Mapping Project

The Reach 11 Trail Mapping Project is a data collecting task I started conducting back in October 2021. My goal for the project is to collect data on all the different trails, trail entrances, markers, info boards, and other various pieces of data into a dataset utilizing the ArcGIS QuickCapture application – and then creating a map or maps of the Reach 11 recreational area. The reasoning behind the project was to learn about ArcGIS’ QuickCapture app and how you can create interactive forms to collect data on the go – I was looking for an application to collect data on Reach 11 and the QuickCapture app was great for that task.

I also wanted to conduct my first real independent project besides the smaller Mapbox projects I had made, to better demonstrate some of my capabilities with geospatial science. And on top of that, I wanted to create my own dataset for public use, with the dataset focused on the trail marker, info boards, and entrances/exits to the various trails. Other pieces of data I collected - such as the trail paths themselves - are already available to the public through services like OpenStreetMaps.

QuickCapture was incredibly useful for this project. Through ArcGIS Online, you’re able to create your own forms for mobile data collection in the field using the QuickCapture app. Besides being able to collect data on-the-go, the accuracy of the data being collected was sound. While not all the data collected will be useful, as some of it is already in the public domain – some pieces of data such as the trail markers and info boards aren’t as readily available for public viewing compared to trail paths or parking lots.

Utilizing Map Viewever Classic, I was able to create an interactive map of the raw data I collected by creating a view-only layer and using that to make my interactive map. This particular map was made for the purpose of being embeded onto my ePortfolio, but I have another map file of the Reach 11 dataset that I utilized and regularly updated while collecting data on the recreational park. This was to help myself validate the data I was collecting - preventing myself from creating multiple entries of the same feature, or thinking that a feature had already been logged when in reality I had actually missed said feature.
